# 使用教程

这一页演示设备控制扩展的典型使用流程：

1. 安装节点程序
2. 让节点连接服务端
3. 在后台确认节点在线
4. 把会话绑定到节点
5. 通过智能体执行设备动作

## 第一步：配置节点连接服务端

### macOS / Windows

打开托盘菜单中的“设置”，填写：

- 服务器地址
- 节点名称（可选，建议填写便于识别的设备名）

保存后，节点会自动注册并建立长连接。

### Linux

如果通过安装脚本传入了 `--server-url`，配置已经写入。也可以手动修改配置文件后重启服务：

```bash
systemctl restart dux-ai-node
```

## 第二步：在后台确认节点在线

进入后台的设备控制页面，确认：

- 节点名称正确
- 状态为在线
- 连接 ID 与设备 ID 已生成
- 延迟与最后在线时间有更新

如果节点已经在线，说明服务端和本地节点通信正常。

## 第三步：把会话绑定到节点

当用户希望在指定设备上执行任务时，智能体会先调用节点列表能力，再把当前会话绑定到目标节点。

绑定完成后：

- 当前会话后续的节点动作会优先在该设备上执行
- 节点托盘中会显示绑定状态
- 后台可查看该节点当前绑定的会话列表

## 第四步：执行常见动作

### 读取系统信息

可以让智能体执行类似任务：

```text
读取当前节点的系统信息
```

常见返回包括：

- 操作系统
- CPU 数量
- 内存情况
- 磁盘信息

### 浏览器读取网页

例如：

```text
打开 https://example.com 并读取正文内容
```

常见动作包括：

- 打开网页
- 抓取标题与正文
- 截图
- 点击元素
- 输入内容

### 文件操作

例如：

```text
列出当前节点的某个目录
读取某个文本文件
```

### 终端命令

例如：

```text
在当前节点执行 git status
```

这类动作通常会进入审批流程，用户确认后才会真正执行。

## 第五步：审批与结果回传

设备控制扩展内置审批机制。

典型流程：

1. 智能体准备执行高风险动作
2. 系统生成审批消息
3. 用户同意或拒绝
4. 节点继续执行或中止
5. 结果回传到会话中

当前常见高风险动作主要是终端命令执行。

## 排障建议

### 节点显示未连接

- 检查服务端地址是否填写正确
- 检查目标机器能否访问服务端
- Linux 查看 `journalctl -u dux-ai-node -f`
- macOS / Windows 查看节点日志与连接日志

### 节点在线但动作下发失败

- 检查会话是否已绑定正确节点
- 检查节点连接 ID 是否有效
- 检查本地节点是否仍在运行
- 检查审批是否已经通过

### Linux 浏览器动作失败

- 检查 Chromium 是否已安装
- 检查当前节点是否允许 headless 浏览器运行
