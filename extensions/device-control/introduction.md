# 设备控制简介

设备控制扩展用于把 Dux AI 的会话能力连接到真实节点设备，让智能体不仅能回答问题，还能在经过授权后读取系统信息、访问文件、执行终端命令、操作浏览器，或者把任务下发到本地节点执行。

当前设备控制扩展由两部分组成：

- `AiDesktop` 模块：负责节点设备注册、绑定、动作下发、结果回传与后台管理
- `dux-ai-node` 节点程序：运行在 macOS、Windows、Linux 设备上，负责本地执行具体动作

## 适合什么场景

- 在指定电脑上执行终端命令、读取日志或检查系统状态
- 让智能体在本地浏览器中打开页面、抓取内容、截图
- 把某个会话绑定到固定节点，让后续动作都在该设备上执行
- 把 Dux AI 扩展到办公电脑、测试机、内网服务器或个人工作站

## 当前能力范围

设备控制扩展当前支持的能力主要包括：

- 节点注册与在线状态上报
- 会话绑定节点
- 浏览器动作
- 文件动作
- 系统信息读取
- 屏幕截图
- 终端命令执行
- 审批与风险控制

其中高风险动作会进入审批流程，例如终端执行命令。低风险动作则可以直接执行，例如读取系统信息、列出目录或抓取网页内容。

## 运行模式

### macOS / Windows

节点以托盘程序运行，适合个人电脑、办公终端或带桌面的设备。

### Linux

节点以 daemon/headless 模式运行，适合 Debian、Ubuntu 等服务器环境，不提供 GUI。

## 与桌面端客户端的区别

- 桌面端客户端：给用户提供聊天和会话界面
- 设备控制扩展：给智能体提供“操作本地设备”的执行能力

两者可以配合使用，但职责不同。
