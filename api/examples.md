# 对接示例

## 示例一：网页聊天窗口

适合官网客服、产品咨询、企业问答。

### 推荐接口组合

1. `GET /agent/v1/models`
2. `POST /agent/v1/sessions`
3. `POST /agent/v1/chat/completions`
4. `GET /agent/v1/sessions/{id}/messages`

### 推荐顺序

- 首次进入页面拉模型列表
- 创建会话后保存 `session_id`
- 发送消息时携带选中的智能体编码
- 切换会话时重新加载消息列表

## 示例二：企业内部系统调用

适合 CRM、OA、项目管理、工单系统。

### 推荐做法

1. 用后台接口预创建智能体
2. 业务系统通过对外 API 发起会话
3. 如果需要执行动作，再接函数能力或工作流

### 适合落地方向

- 客户预约
- 计划生成
- 项目进度查询
- 内部资料问答

## 示例三：机器人平台接入

适合钉钉、飞书、企业微信等聊天入口。

### 推荐做法

1. 后台创建机器人实例
2. 配置平台回调到 `/boot/webhook/{code}`
3. 绑定智能体
4. 在平台发消息验证回复

## 示例四：会话式业务系统

适合电商、外卖、医院等场景。

### 推荐组合

- 智能体负责理解用户意图
- 工作流负责执行业务动作
- 外部系统负责最终业务落库与状态流转

### 适合场景

- 下单与支付引导
- 点餐与外卖业务
- 挂号与预约
- 对话式服务办理

## 示例五：知识库问答入口

适合产品资料查询、企业信息问答、FAQ 助手。

### 推荐组合

- 后台配置知识库与智能体
- 前端仍然通过 `/agent/v1/chat/completions` 发起聊天
- 智能体内部再调用知识库能力完成检索与回答
