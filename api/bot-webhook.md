# 机器人回调接口

机器人平台统一通过以下入口回调：

```http
GET /boot/webhook/{code}
POST /boot/webhook/{code}
```

其中 `{code}` 是机器人实例编码。

## 这条接口做了什么

统一入口会完成：

- 读取机器人实例
- 调用对应平台驱动
- 校验回调合法性
- 解析消息
- 投递到 AI 会话
- 回写结果到平台

## 接入流程

1. 在后台创建机器人实例
2. 复制系统生成的回调地址
3. 配置到钉钉、飞书、QQ 机器人或企业微信平台
4. 绑定智能体
5. 测试消息收发

## 可能出现的失败点

- `code` 对应的机器人不存在
- 机器人未启用
- 平台签名校验失败
- 回调参数不完整
- 智能体链路执行失败

## 推荐排查顺序

1. 先看机器人实例是否存在
2. 再看平台回调参数是否完整
3. 再看智能体是否已绑定
4. 最后再查能力、知识库或工作流日志
