# 服务商扩展

如果你要扩展新的模型平台，重点不是先改页面，而是先明确协议和能力边界。

## 先确认三件事

1. 它是标准 OpenAI 协议，还是私有协议
2. 它支持哪些能力：聊天、Embedding、图片、视频、文件
3. 它的异步任务是否有统一查询接口

## 推荐扩展顺序

### 标准或兼容 OpenAI 协议

优先复用现有 OpenAI Compatible 链路，通常成本最低。

### 图片、视频、Embedding

如果不同能力协议差异明显，建议按能力分别扩展，而不是强行塞进一个实现里。

### 文件解析与上传

如果供应商在文件上传、解析、引用方式上是单独协议，应该作为独立能力链路接入。

## 一个实践原则

先判断能不能复用现有 provider factory，不能复用时再新增协议分支。不要为了“看起来统一”做无意义封装。

## 推荐扩展思路

- 先完成聊天链路
- 再扩 Embedding
- 再扩图片与视频
- 最后再补文件上传、解析和异步查询

这样成本最低，也最容易保证已有功能不被破坏。
