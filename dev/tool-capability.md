# 能力扩展

Dux AI 的能力扩展机制，决定了智能体除了“直接回答”之外还能执行哪些动作。

## 能力来自哪里

能力主要通过注册机制注入到 AI 模块中，常见能力包括：

- 函数调用
- HTTP 请求
- 知识库检索
- 文档解析
- 图片生成
- 视频生成
- 工作流调用
- 异步等待与任务查询

## 一个关键理解

能力配置通常分两层：

- 能力注册：告诉系统“有这个能力”
- 能力实例配置：告诉系统“这个能力具体怎么执行”

所以很多能力在界面里会显示“需配置”，这不是异常，而是正常设计。

## 什么时候用能力，什么时候用工作流

### 直接用能力

适合：

- 单步动作
- 参数少
- 结果简单
- 不需要异步恢复

### 用工作流

适合：

- 多步骤动作
- 需要异步轮询
- 需要节点日志
- 需要结构化最终结果

## 推荐实践

### 面向业务功能

如果你的目标是“让 AI 调一个现有业务动作”，优先考虑函数调用能力。

### 面向外部系统

如果你的目标是“让 AI 调一个 HTTP 接口”，优先考虑 HTTP 请求能力。

### 面向复杂自动化

如果你的目标是“让 AI 完成一整段业务链路”，优先考虑工作流。

## 一个经验原则

不要为了“看起来全都能勾选”就把所有能力都挂到智能体上。优先给每个智能体只保留它真正需要的能力，这样更稳定，也更容易排障。
