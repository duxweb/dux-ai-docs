# 智能体与能力

智能体是 Dux AI 的核心。用户最终面对的不是服务商、模型或向量库，而是这个“能和人交流、还能帮忙办事”的智能体。

![智能体编辑页占位图](/images/agent-form.png)

## 一个智能体通常包含什么

- 名称
- 绑定模型
- 系统提示词
- 能力扩展
- 机器人绑定关系
- 会话与消息记录

## 系统提示词有什么用

系统提示词决定这个智能体是谁、说话风格是什么、回答时优先考虑什么。

例如你可以把它设定成：

- 客服助手
- 企业知识顾问
- 预约受理助手
- 项目管理助理

## 能力扩展是什么

能力扩展的作用，是让智能体不只是“回答”，还能“执行”。

例如：

- 调 HTTP 接口
- 搜知识库
- 解析文档
- 调函数
- 调工作流
- 做图片或视频相关任务

## 为什么有些能力显示“需配置”

因为很多能力勾选以后，还需要你继续补参数。

例如：

- HTTP 请求需要地址和方法
- 函数调用需要默认配置
- 知识库能力需要真正的知识库链路
- 工作流能力需要指定目标流程

所以界面里先勾选能力，只是先把它挂到智能体身上，后面还要继续补配置。
