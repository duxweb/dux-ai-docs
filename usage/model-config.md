# 模型配置

模型配置是服务商下的第二层。服务商决定“连哪家”，模型决定“用哪个”。

![模型配置列表或表单占位图](/images/model-config-overview.png)

## 常见模型类型

- 对话模型
- 图片模型
- 视频模型
- Embedding 模型

## 如何获取模型 ID

这里说的“模型 ID”，本质上就是你在模型配置里要填写的“远端模型名 / 模型标识”。

不同平台叫法可能不一样，有的叫：

- 模型名称
- Model
- Model ID
- 接入点 ID
- Deployment 名称

但在 Dux AI 里，最终都是统一填到模型配置里的对应字段里。

以火山方舟为例进入开通管理并且筛选到语言模型中：

![模型列表](/images/model-config-list.png)

进入详情后可以获得 id 直接复制：

![模型ID](/images/model-config-id.png)

## 第一次应该怎么配

建议按这个顺序：

1. 先配对话模型
2. 如果要做知识库，再补 Embedding 模型
3. 如果要做图片，再补图片模型
4. 如果要做视频，再补视频模型

## 模型表单里哪些最重要

如果你是第一次配置，先重点看这些：

- 服务商
- 模型名称
- 模型 ID
- 模型类型
- 是否启用

其它更高级的参数，可以后面再补。

## 聊天模型里的附件支持怎么理解

聊天模型里有一组“附件能力”配置，主要用来决定：

- 是否支持图片
- 是否支持文件
- 是否支持音频
- 是否支持视频

你可以直接把它理解成：

> 这个聊天模型愿不愿意接收这些附件，以及接收到之后怎么处理。

### 最常见的 3 种策略

- `模型支持`：直接交给模型处理
- `本地解析`：先在本地解析，再把结果交给模型
- `关闭`：不支持这一类附件

## 文件支持会怎么处理

如果聊天模型开启了文件或图片支持，文件上传后通常会有两种情况：

### 1. 模型本身就支持

那就直接交给模型处理。

### 2. 模型不直接支持，但允许解析

那系统会先尝试：

- 直接读取文本型文件
- 或走“解析配置”把图片 / PDF 里的文字识别出来

## 图片、视频和聊天模型的关系

建议直接这样理解：

- 聊天模型：负责理解和对话
- 图片模型：负责图片生成
- 视频模型：负责视频生成

所以通常建议：

- 智能体主模型始终用聊天模型
- 图片、视频能力通过能力或工作流调用

## 媒体存储驱动是做什么的

对于图片和视频模型，后台里还有一个“媒体存储驱动”。

它的作用很简单：

- 图片 / 视频生成后保存到哪里
- 如果不填，就走系统默认存储

所以如果你准备做图片或视频能力，也建议先把“系统存储配置”准备好。

## 文件接口能力需要手工选吗

当前系统会按服务商协议自动识别文件接口能力，不需要你手工去选文件驱动。
