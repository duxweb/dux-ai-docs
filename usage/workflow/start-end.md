# 输入与输出节点

输入节点和输出节点，是工作流里最基础、也最重要的两个节点。

## 输入节点是干什么的

输入节点的作用，就是定义这个工作流一开始可以接收哪些参数。

你可以把它理解成：

> 这是给工作流做“入参表单”的地方。

## 输入节点里可以配什么

根据当前系统能力，输入节点可以配置多种字段类型，例如：

- 文本
- 文本框
- 数字
- 布尔
- 日期
- JSON
- 图片
- 多图片
- 文件
- 多文件

## 标签配置是什么意思

输入节点里除了字段名，还可以配置：

- 展示名称
- 是否必填
- 默认值
- 描述说明

其中“展示名称”就是用户在后台或执行表单里真正看到的标签。

例如：

- 字段名：`prompt`
- 展示名称：`提示词`

所以工作流里的“标签配置”，本质上就是把内部字段变成用户容易理解的名称。

## 第一次怎么配输入节点

建议：

1. 先只保留最少必要字段
2. 每个字段都写清楚展示名称
3. 必填项尽量少一点
4. 默认值只放真正需要的内容

## 输出节点是干什么的

输出节点的作用，就是定义这个工作流最后返回什么。

你可以把它理解成：

> 这是给工作流做“最终结果结构”的地方。

## 输出节点里可以配什么

输出节点主要通过“输出配置”来定义最终字段结构。

你可以控制：

- 返回哪些字段
- 字段名称是什么
- 字段类型是什么
- 每个字段从哪里取值

## 为什么输出节点很重要

如果这个工作流最终还要给：

- 前端页面
- API
- 机器人
- 其他模块

继续使用，那输出节点就最好尽量稳定、结构化。

## 第一次怎么配输出节点

建议：

1. 先只返回最核心的 1~3 个字段
2. 字段名称尽量简单
3. 不要一开始就返回过于复杂的大对象
4. 先确认前端或 API 能稳定消费
