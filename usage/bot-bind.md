# 机器人绑定与排障

平台参数配好之后，还差最后两步：

- 把机器人绑定到智能体
- 验证消息是否真的能走通

![机器人绑定智能体占位图](/images/bot-bind-agent.png)

## 绑定顺序建议

推荐按这个顺序验证：

1. 先确认智能体本身能在后台正常对话
2. 再创建并保存机器人
3. 配好平台回调地址
4. 把机器人绑定到智能体
5. 在平台里发送消息做联调

## 排障顺序

如果机器人收不到消息或不能回复，建议按顺序检查：

1. 回调地址是否正确
2. 平台密钥是否填写完整
3. 机器人是否启用
4. 是否已经绑定智能体
5. 智能体本身是否可正常对话
6. 工作流、能力或知识库是否导致中间执行失败

## 排查时应该先看哪些日志

如果机器人联调有问题，建议优先看下面这几类日志。

### 1. 先看机器人日志文件

默认日志目录在：

```text
data/logs/
```

机器人相关最重要的日志文件是：

```text
data/logs/boot.log
```

这里通常能看到：

- webhook 是否进来了
- 平台回调有没有失败
- 系统最终决定回什么消息
- 平台发送消息时报了什么错

如果这里连请求记录都没有，通常就不是智能体问题，而是平台回调或机器人配置问题。

### 2. 再看智能体日志文件

智能体和桥接链路相关日志通常看：

```text
data/logs/ai.agent.log
```

这里主要看：

- 是否找到了绑定的智能体
- 有没有创建会话
- 有没有用户消息写入
- 有没有生成回复
- 桥接过程中有没有失败

### 3. 如果涉及图片、视频、知识库，再看专项日志

常见日志文件包括：

```text
data/logs/ai.image.log
data/logs/ai.video.log
data/logs/ai.rag.log
data/logs/ai.docs.log
data/logs/ai_scheduler.log
```

大致对应：

- `ai.image.log`：图片生成
- `ai.video.log`：视频生成
- `ai.rag.log`：知识库和检索
- `ai.docs.log`：文件解析
- `ai_scheduler.log`：异步调度、轮询、恢复

## 一个最简单的判断方法

你可以直接按这句话来排查：

> 先看 `data/logs/boot.log` 有没有进来，再看 `data/logs/ai.agent.log` 有没有生成会话和回复，最后再看专项日志是不是出错。

## 推荐测试方式

第一次联调时，不要直接测复杂业务。

建议先发一条最简单的消息，例如：

```text
你好，请介绍一下你自己
```

如果这一步都没通，先不要继续测试知识库、工作流、图片或视频能力。
