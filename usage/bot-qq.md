# QQ 机器人接入

QQ 机器人接入更适合按“注册开放平台账号 -> 实名与主体信息 -> 创建机器人 -> 获取 AppID / App Secret -> 沙箱测试 -> 绑定智能体”的顺序来理解。

## 接入前准备

开始之前，建议先准备：

- QQ 开放平台账号
- 可用的测试 QQ 号
- 一个已可正常工作的智能体

## 1. 注册 QQ 开放平台账号

根据腾讯云开发者社区的接入实践，首先注册并登录 [QQ 开放平台](https://q.qq.com/qqbot/):

![QQ 开放平台入口占位图](/images/bot-qq-console.png)

## 2. 创建机器人

登录开放平台后进入 `机器人` - `创建机器人` 进行创建。

## 3. 获取 AppID 与 App Secret

在 QQ 机器人管理页面 - 管理 - 开发管理中获取：

- `AppID`
- `App Secret`
- `Token`

![QQ 开放平台入口占位图](/images/bot-qq-id.png)

## 4. 消息 URL 配置

在 QQ 机器人管理页面 - 开发 - 回调配置中配置消息回调


![QQ 开放平台入口占位图](/images/bot-qq-webhook.png)

按照图中配置选择消息事件，然后填写 Dux AI 提供的回调地址： `https://域名/boot/webhook/机器人编码`。


## 4. 沙箱测试

QQ 机器人接入通常很适合先走沙箱测试，例如：

- 把自己的 QQ 号加入测试白名单
- 先在沙箱里单聊测试
- 再继续接群聊或更复杂场景

## 5. 回到 Dux AI 配置

在 Dux AI 当前机器人配置里，对应常见字段包括：

| 配置项 | 说明 | 是否重点 |
| --- | --- | --- |
| `App ID` | QQ 机器人应用 ID | 是 |
| `App Secret` | QQ 机器人应用密钥 | 是 |
| `回调 Token` | 回调 Token | 按需 |

## 6. 绑定智能体并联调

然后再把机器人绑定到智能体，进行联调。

### 推荐测试顺序

1. 先做沙箱单聊测试
2. 再测更复杂场景
3. 最后再接知识库、工作流或多媒体能力

## 常见问题

### 沙箱里能收到消息，正式环境不行

优先检查：

- 是否已经通过平台审核或发布
- 回调参数是否切换成正式环境配置
- 机器人实例是否对应正确环境
