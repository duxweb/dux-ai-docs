# 知识库检索能力


适合让智能体在回答前先去查资料，而不是只靠模型自己的通用知识。

## 适合什么场景

- 企业知识问答
- 产品文档查询
- FAQ 回答
- 内部制度说明

## 需要先准备什么

在使用这个能力之前，建议先准备好：

- 一个知识库
- 可用的向量库
- 可用的 Embedding 模型
- 知识库里已经有内容可检索

## 后台里通常要配什么

根据当前能力定义，这个能力最主要的配置项有：

### 知识库

选择要检索的目标知识库。这一项是必填。

### 返回条数

决定一次最多取多少条检索结果。

- 默认值：`5`
- 常见建议：先保持默认

### 内容长度

决定检索结果里的正文是否截断。

- `0` 表示不截断
- 如果你担心返回内容太长，可以设置一个合适长度

## 使用建议

- 第一次使用时，先用默认返回条数 `5`
- 如果回答不够准，优先检查知识库内容和检索效果，而不是先怪模型
- 如果结果太长，再调“内容长度”

## 推荐的最小配置

第一次使用时，建议先这样配：

- 选一个已经建好的知识库
- 返回条数保持 `5`
- 内容长度先用 `0`（不截断）

这样最容易先看清真实检索效果。
