# 通知发送能力


适合把智能体或流程的结果进一步发到外部通知渠道。

## 适合什么场景

- 系统通知
- 业务提醒
- 流程完成通知
- 外部消息推送

## 后台里通常要配什么

### 通知通道

当前主要支持：

- Boot 会话消息

### 机器人编码

这是必填项，决定通知通过哪个机器人发出去。

### 标题

- 默认值：`提醒通知`
- 不填时会用默认标题

### 通知内容

正文内容，可与图片二选一，也可以同时填写。

### 图片链接

如果你想发图片通知，可以填写图片 URL。

## 使用建议

如果你的通知内容很简单，这个能力就足够；如果你的通知还要串很多业务步骤，更建议放进工作流。

## 推荐的最小配置

第一次使用时，建议先这样配：

- 通知通道用默认值
- 选一个机器人
- 只填标题和正文
- 先不带图片

先把一条最简单通知发通。
