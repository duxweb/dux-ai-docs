# HTTP 请求能力


适合让智能体主动调外部 HTTP 接口。

## 适合什么场景

- 查订单状态
- 查预约信息
- 调业务系统接口
- 推送或同步外部数据

## 后台里通常要配什么

### 请求方法

支持：

- GET
- POST
- PUT
- PATCH
- DELETE

### 请求 URL

这是最核心的配置项，必须填写。

### Query 参数

适合放查询字符串参数。

### Headers

适合放认证头、业务头等。

### Body 类型

支持：

- JSON
- Form Data
- Raw

### Body 内容 / Form Data

- 如果你选 JSON 或 Raw，就填 `Body 内容`
- 如果你选 Form Data，就填 `Form Data`

### 请求超时

- 默认值：`30 秒`
- 建议不要设置得过大

## 使用建议

- 第一次先用 GET 跑通
- 再加认证 Header
- 再处理复杂 Body
- 不建议直接暴露危险写接口给智能体

## 推荐的最小配置

第一次使用时，建议先这样配：

- 请求方法：`GET`
- 请求 URL：先填一个可稳定访问的测试接口
- 不先加复杂 Header
- 请求超时保持默认 `30 秒`

先确认能正常返回，再逐步加鉴权和复杂 Body。
