# MCP 能力

MCP 能力适合把更多外部工具和服务接入智能体。

![MCP 能力配置占位图](/images/agent-mcp.png)

## MCP 的价值是什么

MCP 的价值不是“多接一个工具”，而是让 Dux AI 的智能体和工作流，变成一个可以持续扩展能力边界的平台。

换句话说：

- 没接 MCP 前，智能体主要依赖系统内置能力
- 接入 MCP 后，智能体和工作流就可以继续接更多外部工具、外部服务和行业能力

这意味着 Dux AI 不再只是“会回答”，而是可以越来越“会做事”。

## 适合什么场景

- 接入外部工具
- 扩展智能体执行能力
- 接第三方服务或平台能力
- 给工作流补更多动作能力
- 把现有业务系统和智能体能力继续打通

你可以通过 MCP，把智能体和工作流扩展成几乎无限增长的能力平台，例如：

### 业务系统类

## 当前支持的连接方式

根据现有实现，当前已支持：

- `streamable_http`
- `sse`

## 后台里通常要配什么

### MCP 工具名

这是必填项，填写远端 MCP Server 暴露出来的工具名。

### MCP 服务 URL

这是必填项，填写 MCP 服务地址。

### 传输方式

一般按服务端支持的方式选择：

- `streamable_http`
- `sse`

### 请求头

适合放鉴权信息。

### Token

如果服务端要求 Token，可以直接填在这里。

### Arguments（入参）

这个能力真正要传给 MCP 工具的参数，不是在上面这些配置项里直接逐个填写，而是放在调用时的 `arguments` 里。

你可以直接理解成：

- 上面的配置项：决定“连哪个 MCP 服务、调用哪个工具”
- `arguments`：决定“这次调用这个工具时具体传什么参数”

### `arguments` 支持什么格式

当前实现里，`arguments` 支持：

- `对象`
- `JSON 字符串`

例如：

```json
{
  "arguments": {
    "query": "Dux AI 是什么？"
  }
}
```

或者：

```json
{
  "arguments": "{\"query\":\"Dux AI 是什么？\"}"
}
```

如果 `arguments` 里的某个值本身还是一段 JSON 字符串，系统也会继续尝试解析。

## 推荐可以接的在线 MCP 平台

如果你不想自己搭 MCP 服务，可以优先关注已经提供在线工具能力的平台。

### MCP-GET

`mcp-get.com` 更适合当作 MCP 服务目录或发现平台来使用。它更像“找 MCP 服务”的入口，而不是默认直接填写到服务 URL 的统一服务本身。

建议入口：

- `https://mcp-get.com`

适合：

- 找可用 MCP 服务
- 看有哪些工具生态可接
- 选择后再接具体 MCP 服务地址

### 魔搭（ModelScope）

如果你希望接入更多现成 AI 工具、模型能力或推理服务，可以优先关注魔搭生态。

建议入口：

- `https://www.modelscope.cn/`

适合场景：

- 快速接入更多模型工具
- 做实验性能力扩展
- 补充智能体和工作流的外部执行能力

### 其他在线 MCP 服务

如果第三方服务本身已经提供 MCP Server，也可以直接接入它的在线地址。

选择时建议优先看：

- 是否稳定可访问
- 是否支持 `streamable_http` 或 `sse`
- 是否有清晰的工具名和调用说明
- 是否支持认证头或 Token

## 推荐的最小配置

第一次使用时，建议先这样配：

- MCP 工具名填一个最简单的工具
- 服务 URL 填一个稳定可访问的地址
- 传输方式先用默认支持的方式
- `arguments` 先只传一个最简单的对象

先确认一项工具能通，再继续加更多 MCP 工具。
