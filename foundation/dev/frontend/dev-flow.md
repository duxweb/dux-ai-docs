# 前端开发流程

这份说明回答两个最常见的问题：

- 什么时候需要构建前端？
- 为什么页面改了能直接生效？

## 为什么改完就生效

后台页面是“模块页面”，放在 `app/模块/Admin` 目录中。前端基座按需加载这些页面，因此 **无需为每个页面单独编译**。

## 什么时候需要构建

- **开发调试**：通常只需要启动 PHP 服务，页面改完刷新即可。
- **升级前端基座**：当你更新 `web/` 目录下的前端基座代码时，需要重新构建。

## 常见问题

- 访问 `/manage/` 白屏：检查是否已构建基座，或 `vite.dev` 是否正确配置。
- 页面不更新：确认页面路径与菜单 loader 是否一致。

## 推荐工具（pnpm）

```bash
cd <项目目录>
pnpm install
pnpm build
```
