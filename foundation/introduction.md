# 基础框架概览

这一部分讲的是 Dux AI 的基础框架，也就是 `Dux PHP Admin` 本身。

如果你现在关注的是智能体、知识库、机器人和工作流，请先看 [使用概览](/usage/introduction)；如果你想先理解这套系统下面的中后台、模块化和资源接口是怎么组织的，就从这里开始。

## Dux AI 和基础框架是什么关系

你可以把当前主站理解成两层：

- `Dux PHP Admin`：中后台底座，负责路由、资源、权限、菜单、配置、系统模块和部署运行
- `Dux AI`：在这套底座之上继续扩展智能体、机器人、知识库、工作流和对话能力

所以 Dux AI 不是脱离后台框架重新造的一套系统，而是在同一套模块体系中继续扩展 AI 能力。

## 与 Dux Lite / DVHA 的关系

- `Dux Lite`：后端基座，提供路由、资源、权限、命令行、配置与数据库能力
- `DVHA`：前端基座，负责管理端路由、布局与数据交互
- `Dux PHP Admin`：把两者组合成一套完整后台框架

## 与常见方案的差异

| 对比项 | Dux PHP Admin | 传统模板后台 | 传统前后端分离 |
| --- | --- | --- | --- |
| 页面组织 | 页面放在模块里，按需加载 | 往往以模板文件为主 | 页面和后端工程拆开 |
| 接口组织 | 资源化接口自动生成 | 常见是手写控制器和权限 | 两端单独维护接口契约 |
| 权限与菜单 | 菜单、权限、路由天然联动 | 常见需要额外拼装 | 需要两端再做一层映射 |
| 开发体验 | 接近传统全栈，但保留现代 Vue 体系 | 上手快但前端栈偏旧 | 规范强，但协作成本更高 |

## 核心组成

- `System` 模块：用户、角色、菜单、配置、日志、存储、调度等基础后台能力
- `Install` 模块：安装向导和首次初始化流程
- `Data` 模块：数据管理和配置化能力
- `web/`：后台前端源码
- `config/`：应用、数据库、运行配置

## 路由与访问入口

- 管理界面入口：`/manage/`
- 后台 API：`/admin/*`
- 开放 API：`/api/*`

> `/manage` 会自动跳到 `/manage/`，文档里统一使用带斜杠的地址。

## 默认模块

当前基础框架默认注册的模块来自 `config/app.toml`：

```toml
registers = [ "App\\System\\App", "App\\Install\\App", "App\\Data\\App" ]
```

你可以继续添加自己的模块，并在这里注册。

## 目录概览

```text
app/
  System/          # 系统模块
  Install/         # 安装模块
  Data/            # 数据模块
config/
  app.toml         # 注册模块
  database.toml    # 数据库
  use.toml         # 运行配置
public/
  index.php        # 入口
  static/web/      # 后台前端构建产物
web/               # 后台前端源码
```
